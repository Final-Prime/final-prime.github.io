Capability / Deliberately narrow

Built for an exact, bounded job.

Prime Compression is presented as the system that can be evaluated now, not as a universal compressor or a completed streaming platform.

01 / Compile

Offline archive

A private operator compiles an agreed, bounded asset set into a delivery archive for the evaluated runtime.

02 / Decode

Minimal runtime

A source-free runtime verifies and reconstructs the approved output set without distributing the private operator or source.

03 / Verify

Exact parity

Extraction is accepted only when every delivered output matches the declared reference hashes byte for byte.

Evaluation / Customer holdout

Freeze first. Measure second.

The evaluation is designed to make a negative result valid. If Prime Compression is not the right fit for the supplied workload, the report should say so.

  1. 01
    Qualify the workload

    Define formats, corpus size, target platform, current baseline and the constraint that matters most.

  2. 02
    Freeze the candidate

    Record the exact operator, runtime, test plan and acceptance rules before inspecting the holdout.

  3. 03
    Run the private holdout

    Compile and decode the customer-supplied sample in a controlled environment without exposing protected internals.

  4. 04
    Compare the full delivery

    Measure archive, runtime, metadata, latency, peak memory and exact reconstruction against agreed baselines.

  5. 05
    Issue the decision

    Return an evidence report with limitations and a clear no-fit, further-test or scoped-pilot recommendation.

Full byte accounting / Required

The runtime counts.

An archive-only ratio can hide the cost of the decoder and metadata. The evaluation reports the complete first-delivery footprint and records the environment used for performance measurements.

  • Archive bytes
  • Runtime binary bytes
  • Manifest and metadata bytes
  • Decode p50 and p95
  • Peak memory
  • Exact hash parity

Claim boundary / Explicit

What this surface says, and what it does not.

The public description stays narrower than the protected implementation and narrower than any future product ambition.

Asserted here

In scope
  • Exact reconstructionThe approved output must reproduce byte for byte.
  • Offline archive compilationThe current target is a bounded, precompiled asset set.
  • Minimal runtime decodingA source-free decoder is the intended evaluation delivery surface.
  • Private customer evaluationQualification, written scope and a representative holdout are required.

Not asserted here

Out of scope
  • No general zstd or xz superiorityAny advantage must be established for the evaluated workload.
